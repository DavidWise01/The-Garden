# 40D Final — Bipolar Dimensions + Observer Stack

## Core form

The system is not a 4-domain Cartesian model. It is a **bipolar dimensional field** with an attached observer stack:

- **+20 D**: constructive / generative dimensions
- **-20 D**: inverse / entropic dimensions
- **20 O**: observer dimensions

Total addressable axes: **60**

A state is expressed as:

```text
State = { plus: P_i, minus: N_j, observer: O_k, polarity, magnitude, phase }
```

Where:
- `P_i` is one of the 20 positive dimensions
- `N_j` is one of the 20 negative dimensions
- `O_k` is one of the 20 observers
- `polarity` can be `positive`, `negative`, or `balanced`
- `magnitude` is a scalar intensity
- `phase` is a cycle position for animation / convergence

## Engine model

The executable kernel uses:
- one active positive dimension
- one active negative dimension
- one active observer
- dynamic magnitude and phase
- projection modes for rendering

### Main operators
- `shiftPlus()`
- `shiftMinus()`
- `shiftObserver()`
- `invert()`
- `balance()`
- `collapseToSeed()`
- `converge()`
- `linkHarvesterFacet()`

## Projection model

Since 60 axes cannot be directly rendered, the visualizer uses:
- **Tri-layer view**: positive ring, observer ring, negative ring
- **Chord view**: links between active positive / observer / negative nodes
- **Slice view**: selected dimension families
- **Harvester bridge view**: overlays source facets from the inverse harvester
